Введение

"Возникает вопрос: зачем использовать go run, если есть go build?"

"Разница проста. go build создаёт постоянный исполняемый файл, а go run — нет."

Как работает go run

Процесс:

"Команда go run в Go компилирует и запускает программу за один шаг, не создавая постоянный бинарный файл."

"При выполнении go run Go сначала компилирует исходный код .go во временный исполняемый файл, который хранится в оперативной памяти (RAM)."

"После успешной компиляции Go сразу запускает скомпилированную программу."

Ключевые особенности:

Временный файл удаляется после выполнения, не сохраняясь на диске.

Идеально для быстрого тестирования и запуска небольших программ во время разработки.

Как работает go build

Процесс:

"go build создаёт постоянный бинарный исполняемый файл на диске."

Пример: выполнение go build hello.go генерирует hello.exe (Windows) или hello (Unix-системы).

Ключевые особенности:

Бинарный файл можно запускать многократно без перекомпиляции.

Используется для деплоя, тестовых сред или распространения программ.

Основные различия

go run	                        go build
Компиляция + запуск в один шаг.	Создание автономного бинарного файла.
Не оставляет файлов на диске.	Сохраняет исполняемый файл на диске.
Временный файл хранится в RAM.	Файл сохраняется в хранилище.
Для разработки и тестирования.	Для финального деплоя.
Сценарии использования

Когда использовать go run:

На этапах разработки или тестирования, когда код часто меняется.
Пример: "При создании API или ПО вы постоянно вносите изменения и тестируете их. Постоянная пересборка бинарных файлов была бы неэффективной."

Когда использовать go build:

Когда программа готова к деплою или распространению.
Пример: "Скомпилируйте один раз, сохраните бинарный файл и запускайте его где угодно без перекомпиляции."

Лучшие практики

Используйте go run для:

Быстрой итерации и тестирования.

Избежания «захламления» временными файлами.

Используйте go build для:

Создания готовых к продакшену исполняемых файлов.

Распространения программ без исходного кода.

Заключение

"go run упрощает разработку, объединяя компиляцию и запуск в один шаг. go build подготавливает программу к реальному использованию."

"Выбирайте go run для гибкости во время кодинга, а go build — когда готовы к релизу."
