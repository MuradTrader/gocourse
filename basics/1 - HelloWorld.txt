Создание файла

Имя файла:
"Начнём с названия файла. Давайте назовём его hello.go."

"В Go файлы используют расширение .go, аналогично .js для JavaScript."

Структура программы

Объявление пакета:
"Первое, что мы делаем в Go, — объявляем пакет. Для исполняемой программы это должен быть package main."

"Пакет main — это точка входа в программу на Go, независимо от её размера."

Импорт пакетов:
"Через import мы подключаем внешние пакеты. 

Здесь мы импортируем fmt (format package) для операций ввода-вывода, таких как Println."

Функция main:
"func main() — это точка входа любой программы на Go. Без неё программа не запустится."

Демонстрация:
Если заменить func main() на func root(), возникнет ошибка.

"Компилятор ищет func main() — нет main, нет выполнения."

Запуск программы
Команды в терминале:

Рекомендация: используйте go run hello.go вместо ярлыков IDE (например, кнопки запуска в VS Code).

Причина: "Ручной ввод команд помогает лучше понять процесс, особенно новичкам."

Выполнение кода:
Пример: запуск go run hello.go выведет "Hello World" через fmt.Println("Hello World").

Примечание: Go поддерживает Unicode, что позволяет выводить текст на разных языках.

Ключевые концепции

Пакет main:
"Пакет main указывает на автономный исполняемый файл. Компилятор ожидает здесь функцию main."

Демонстрация ошибки: если заменить package main на package root (даже с func main()), программа не запустится.

Сборка исполняемых файлов:
Используйте go build hello.go, чтобы скомпилировать бинарный файл.

Преимущества:
Бинарный файл (например, hello.exe) может запускаться самостоятельно без перекомпиляции.

Запуск через ./hello (или hello.exe в Windows) даёт тот же результат, что и go run.

Демонстрации ошибок
Отсутствие функции main:

Ошибка: runtime.main_main·f: function main is undeclared in the main package.

Некорректное имя пакета:

Ошибка: package command-line-arguments is not a main package.

Лучшие практики
Всегда начинайте с package main и func main().

Используйте go build для создания готовых к продакшену бинарных файлов, 
чтобы избежать повторной компиляции.
